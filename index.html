🚀 Concepts of Git & GitHub

👨‍💻 Shaheer Uddin Ahmed

🔧 What is Git?

Git is a distributed version control system used to track changes in source code during software development.

  • 🌀 A distributed version control system
  • 🌿 Tracks source code changes
  • 📜 Supports team collaboration

🌐 What is GitHub?

GitHub is a cloud-based hosting service for Git repositories. It adds collaboration tools like issues, pull requests, and wikis.

  • ☁️ Host remote repositories [cloud-based]
  • 👥 Features: PRs, Issues, CI/CD
  • 🔓 Open-source and private projects

🌐 Common Git Terms

  • 📁 Repository: A code folder (project container)
  • 🌿 Branch: A separate line of development
  • 📥 Clone: Local copy of a repository
  • 💾 Commit: A saved change with a message
  • 🔀 Merge: Combining branches or changes
  • 📤 Push / 📥 Pull: Sync with remote GitHub

📜 Common Git Commands


git init             # 🆕 Start a repo
git status           # 📋 View status
git add .            # ➕ Stage all changes
git commit -m "msg"  # 📝 Commit with a message
git push origin main # 🚀 Push to GitHub
git pull             # ⬇️ Sync changes

🔄 Typical GitHub Workflow

  1. 🔍 Clone the repository
  2. 🌱 Create a new branch
  3. 🧠 Make changes and commit
  4. 🚀 Push to GitHub
  5. 📬 Open a Pull Request
  6. ✅ Merge to `main`

🌐 How teams use Github

  • 📬 Pull Requests (PRs): Propose and discuss changes before merging
  • 🕵️‍♂️ Code Review: Collaboratively review code for quality and issues
  • 🤖 Automation and CI/CD: Automatically build, test, and deploy code

✅ Best Practices

  • 🧾 Write clear commit messages
  • 🌿 Use branches per feature
  • 🔄 Pull frequently to stay in sync
  • 🔍 Review code before merging
